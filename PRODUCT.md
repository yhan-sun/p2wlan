# Product

## Register

product

## Users

需要在 macOS、Windows 和 Linux 设备之间建立私有虚拟内网的个人用户、开发者和自托管管理员。他们通常在排查连接、启动系统网卡或确认远端设备是否在线，界面必须让非网络专家也能判断当前状态和下一步操作。

## Product Purpose

P2WLAN 提供低内存、跨平台、系统 API 优先的 P2P 虚拟内网。产品成功意味着用户登录控制面并完成一次明确的系统授权后，可以可靠启动 TUN、看到真实连接路径和设备状态、在直连失败时自动使用中继，并能从桌面窗口或系统托盘安全停止和退出。

## Brand Personality

克制、可信、原生。界面像成熟的系统网络工具，状态清楚、动作可预测，不使用营销式表达或装饰性视觉噪声。

## Anti-references

不要做成卡片堆叠的 SaaS 仪表盘、霓虹或紫色渐变的网络工具、展示无法执行的假开关，也不要用持续旋转和模糊文案掩盖后台真实阶段。离线、未知和零值必须有不同含义。

## Design Principles

1. 真实状态优先：每个标签和数字必须来自明确的数据源。
2. 系统动作可观察：授权、启动、注册、建网、停止和失败都有独立阶段与反馈。
3. 默认路径简单：主界面只保留连接、设备、设置和诊断的核心任务。
4. 后台行为可控：关闭窗口、保留隧道和完全退出之间没有隐式副作用。
5. 低资源常驻：共享状态、合并请求、窗口隐藏时降频，不为装饰引入持续负载。

## Accessibility & Inclusion

以 WCAG 2.1 AA 为基线。状态不能只依赖颜色，所有操作支持键盘和清晰焦点，动态状态使用可读文本并尊重减少动态效果设置。
